Overview

Welcome! Here you will find a tutorial to familiarize yourself with the Chemelt workflow!

1) Data loading and preparation
  1. In this tutorial we will work with some example data. Please download the NanoDSF example data nDSFdemoFile.xlsx to your computer. This data is exemplary data of a NanoDSF experiment. It contains multiple thermal denaturation curves of samples with different chemical denaturant concentrations of two denaturants, GuHCl and Urea.
  2. In the tab "1. Import data" upload the data in the Input field under "1. DSF files". You can upload multiple files at once. The tool can handle different file formats (check the User Guide).

  3. The option "Signal" let's you choose different signals captured in the experiment and present in the file. Here we choose 350nm. Additionally, the signals can be normalised. The option "Temperature range (ºC)" let's you choose the temperature range which will be analyzed.

  4. In the field "[Denaturant] (M)" you can choose which denaturant concentrations should be included in the fitting. As the demo file contains the results of two kinds of denaturants, GuHCl and Urea, we choose only the samples with GuHCl.

  5. In the field underneath we can see the data as a plot of the signal against temperature.

  6. As we can see here, the curve with the denaturant concentration of 6.54 M and 2.5 M (uppermost curve) are outliers and are thus excluded from the analysis in the "[Denaturant] (M)" field.

2) Function fitting
  1. Under the tab "2. Fitting" we can set the settings for the fitting of the formula to the curves.

  2. We set the windows for the baselines as 10 degrees Celsius in 2a and 2b each. They control the temperature window sizes at the beginning and end of the curves which are used for the initial estimation of the baseline functions of the native and unfolded protein and shouldn't be too narrow.

  3. The Setting 2c controls the amount of parameters and type of fitting. For more information check out "Choosing a model" in the User Guide. We choose the setting global slopes and global intercepts as this method has the least parameters. The "Fit scale factor" corrects for small experimental errors. The Number of residues is necessary for an initial estimate of the ΔCp parameter and is the number of residues of the analyzed protein.

  4. The Setting 2d and 2e control the type of baseline fitted to the curves. For the native baseline (2d) a linear baseline is usually used and for the unfolded baseline (2e) a quadratic. Here we use linear for the native and exponential for the unfolded, as we found that the exponential baseline generalizes better. The option "Set param bounds" let's you manually choose the parameter boundries and the option "Fit data subset" let's you reduce the number of datapoints for a faster fitting with often a very similar result and thus we choose 100 maximal points per curve. With the "Run Fitting!" button the fitting is started. This can take some minutes depending on the size of the dataset.

3) Analysis
  1. The fitted data can be seen in the "Fitted signal" Tab as black lines.

  2. Underneath the graph are different plotting options to change the visualisation.

  3. In the Tab "Fitted params" you can find the thermodynamic parameters as well as technical parameters used to fit the formula to the curve with their respective units. Additionally, you can find the relative error and the boundries used for fitting. The relative error is an indication of the quality of the fit. While a high relative error indicates an inadequate model, a low error is not proof of an adequate one.
    The resulting thermodynamic parameters should always be critically evaluated, as errors of the fit are possible and they should roughly match expectations. If they reach the boundaries, for example, it can be due to outliers not being accounted for, which skew the results.

  4. In the tab "DG vs T" you can find the calculated Gibbs free energy ΔG againt the temperature as a graphical representation.

  5. The results can be exported in the tab "3. Export results". Here the fitted parameters, the original signal and the fitted signal can be exported and downloaded as a CSV file for further analysis.
